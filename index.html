GIT

  • ls -al ~/.ssh: command to check for existing SSH key
  • cat ~/.ssh/id_ed25519.pub: command for opening existing SSH key
  • cat ~/.ssh/id_ed25519.pub | clip: command for copying existing SSH key
  • notepad ~/.ssh/id_ed25519.pub | clip: command for opening SSH key in notepad
  • ssh -T git@git.bootcampcontent.com: command for verifying SSH key has been setup in Git account and connection has been established
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: check the branch adding the changed files to the staging area
  • git commit -m: "added CSS to project": Commit the changes to repository with description message
  • git pull origin main: make sure current branch is up to date by pulling latest version of Main branch before opening a pull request
  • git push origin branch-name: push changes to Github remote server

Javascript

  • A variable is a named container that allows us to store data in our code.
  • var (method supported on older browswer/not recommended anymore): allows variable to declared after initialisation and can be declared multiple times. Variables created with var have global scope whereas let and const are created as local variable by default.
  • let: can only be declared once and declaration need to come before initialisation. Object data can be updated.
  • const: variable must be declared and initialised at the same time. The object assigned to a const variable cannot be changed or updated, but the object's property value can be updated.
  • Control flow is the order in which a computer executes code in a script.
  • JS is interpreted in order of code and may fail if DOM hasn't completed loading
  • Async loading <script async src="js/script2.js"></script>: scripts will execute as soon as the download is complete. This blocks the page and does not guarantee any specific execution order. Use if your scripts should be run immediately and they don't have any dependencies
  • Defer loading <script defer src="js/script2.js"></script>: scripts will load in the order they are in and will only execute once everything has finished loading.If your scripts need to wait for parsing and depend on other scripts and/or the DOM being in place, load them using defer and put their corresponding </script> elements in the order you want the browser to execute them.
